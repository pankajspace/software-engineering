# Docker Main Concepts with Examples and Commands

Docker is a platform that allows developers to package, ship, and run applications in containers. These containers are lightweight and portable, ensuring consistency across environments, whether in development or production. In this guide, we'll explore Docker's core concepts, with practical examples and commands for each.

---

## 1. Docker Images

### Concept:
- A Docker image is a read-only template that contains the application code, dependencies, runtime, and system libraries required to run a container. 
- Images are used to create Docker containers.

### Commands:
- List Images:

  ```bash
  docker images
  ```

- Pull an Image from Docker Hub:

  ```bash
  docker pull <image-name>
  ```

  Example:

  ```bash
  docker pull nginx
  ```

- Build an Image:

  ```bash
  docker build -t <image-name> <path>
  ```

  Example:

  ```bash
  docker build -t my-node-app .
  ```

  This builds a Docker image using the Dockerfile in the current directory and tags it as `my-node-app`.

- Remove an Image:

  ```bash
  docker rmi <image-id>
  ```

- Remove All Unused Images:

  ```bash
  docker image prune
  ```

### Example:
```Dockerfile
# Dockerfile for a Node.js app
FROM node:14

# Create app directory
WORKDIR /usr/src/app

# Install app dependencies
COPY package*.json ./
RUN npm install

# Copy app source code
COPY . .

# Expose the app's port
EXPOSE 8080

# Start the application
CMD ["node", "app.js"]
```

To build the image from this Dockerfile:
```bash
docker build -t node-app .
```

## 2. Docker Containers

### Concept:
- A container is a runnable instance of a Docker image. Containers are isolated environments that can run applications and services independently of the host system.
  
### Commands:
- Run a Container:

  ```bash
  docker run -d -p <host-port>:<container-port> --name <container-name> <image-name>
  ```

  Example:

  ```bash
  docker run -d -p 8080:80 --name webserver nginx
  ```

  This command runs the `nginx` container in detached mode, mapping port `80` inside the container to port `8080` on the host machine.

- List Running Containers:

  ```bash
  docker ps
  ```

- List All Containers (including stopped):

  ```bash
  docker ps -a
  ```

- Access a Running Container's Shell:

  ```bash
  docker exec -it <container-id or container-name> /bin/bash
  ```

  Example:

  ```bash
  docker exec -it webserver /bin/bash
  ```

- Stop a Running Container:

  ```bash
  docker stop <container-id or container-name>
  ```

  Example:

  ```bash
  docker stop webserver
  ```

- Stop All Running Containers:

  ```bash
  docker stop $(docker ps -q)
  ```

- Remove a Stopped Container:

  ```bash
  docker rm <container-id or container-name>
  ```

- Remove All Stopped Containers:

  ```bash
  docker container prune
  ```

### Example:
```bash
docker run -d -p 3000:3000 --name my-node-app node-app
```

This runs the previously built `node-app` image and maps port `3000` in the container to port `3000` on the host.

## 3. Docker Volumes

### Concept:
- Volumes are used to persist data generated by Docker containers. Docker containers are ephemeral, meaning their data is lost when the container stops. Volumes provide a way to store this data outside the container.

### Commands:
- Create a Volume:

  ```bash
  docker volume create <volume-name>
  ```

  Example:

  ```bash
  docker volume create my-volume
  ```

- List Volumes:

  ```bash
  docker volume ls
  ```

- Mount a Volume to a Container:

  ```bash
  docker run -d -p <host-port>:<container-port> -v <volume-name>:/path/in/container <image-name>
  ```

  Example:

  ```bash
  docker run -d -p 8080:80 -v my-volume:/usr/share/nginx/html nginx
  ```

  This mounts `my-volume` at `/usr/share/nginx/html` inside the container.

- Inspect a Volume:

  ```bash
  docker volume inspect <volume-name>
  ```

- Remove a Volume:

  ```bash
  docker volume rm <volume-name>
  ```

- Remove All Unused Volumes:

  ```bash
  docker volume prune
  ```

### Example:
```bash
docker run -d -p 8080:80 -v my-html:/usr/share/nginx/html nginx
```

Here, `my-html` is a volume that stores files outside the container, and it's mounted into the container at `/usr/share/nginx/html`.

## 4. Docker Networks

### Concept:
- Docker networks allow containers to communicate with each other. Containers on the same network can easily talk to each other via container names.

### Commands:
- Create a Network:

  ```bash
  docker network create <network-name>
  ```

  Example:

  ```bash
  docker network create my-network
  ```

- Run a Container on a Specific Network:

  ```bash
  docker run -d --network <network-name> <image-name>
  ```

  Example:

  ```bash
  docker run -d --network my-network nginx
  ```

- Connect an Existing Container to a Network:

  ```bash
  docker network connect <network-name> <container-id>
  ```

- List Networks:

  ```bash
  docker network ls
  ```

- Inspect a Network:

  ```bash
  docker network inspect <network-name>
  ```

- Remove a Network:

  ```bash
  docker network rm <network-name>
  ```

- Remove All Unused Networks:

  ```bash
  docker network prune
  ```

### Example:
```bash
docker network create backend-network
docker run -d --network backend-network --name db mongo
docker run -d --network backend-network --name app node-app
```

This example creates a network called `backend-network`, runs a MongoDB container on it, and a Node.js application container, allowing both to communicate through the network.

## 5. Docker Compose

### Concept:
- Docker Compose is a tool for defining and running multi-container Docker applications. With a `docker-compose.yml` file, you can define services, networks, and volumes for your application in one file.

### Commands:
- Start Services:

  ```bash
  docker-compose up
  ```

- Start Services in Detached Mode:

  ```bash
  docker-compose up -d
  ```

- Stop Services:

  ```bash
  docker-compose down
  ```

- Build and Start Services:

  ```bash
  docker-compose up --build
  ```

### Example `docker-compose.yml` for Node.js and MongoDB:

```yaml
version: '3.8'

services:
  app: # Node.js application service
    build: . # Build the app from the local directory
    ports:
      - "3000:3000"
    environment: # Environment variables
      MONGO_URI: mongodb://mongo:27017/mydb
    depends_on: # Wait for the MongoDB service to be ready
      - mongo

  mongo: # MongoDB service
    image: mongo:4.4 # Use the MongoDB image from Docker Hub
    ports:
      - "27017:27017"
    volumes:  # Named volume for persistent storage
      - mongo-data:/data/db # Mount the volume to the MongoDB data directory

volumes: # Define named volumes
  mongo-data: # Named volume for MongoDB data
```

This example defines two services:
- app: A Node.js application, built from the local directory, which connects to the MongoDB container.
- mongo: A MongoDB service with a named volume `mongo-data` for persistent storage.

Run the services using:
```bash
docker-compose up
```

## 6. Dockerfile

### Concept:
- A `Dockerfile` is a script that contains instructions for building a Docker image. It defines the base image, copies application files, installs dependencies, and sets up the environment for the application to run.

### Example Dockerfile for Node.js:

```Dockerfile
# Use an official Node.js runtime as a base image
FROM node:14

# Set the working directory inside the container
WORKDIR /usr/src/app

# Copy the package.json and package-lock.json to install dependencies
COPY package*.json ./
RUN npm install

# Copy the rest of the application
COPY . .

# Expose port 3000 to the outside world
EXPOSE 3000

# Start the Node.js app
CMD ["node", "app.js"]
```

To build an image from this Dockerfile:
```bash
docker build -t node-app .
```

## 7. Docker System Management

### Concept:
- Docker provides commands to manage and clean up system resources like unused images, containers, networks, and volumes.

### Commands:
- Remove All Stopped Containers:

  ```bash
  docker container prune
  ```

- Remove All Unused Images:

  ```bash
  docker image prune
  ```

- Remove All Unused Volumes:

  ```bash
  docker volume prune
  ```

- Remove All Unused Networks:

  ```bash
  docker network prune
  ```

- Remove All Unused Data (containers, volumes, networks, images):

  ```bash
  docker system prune
  ```

---

## Conclusion

Docker enables developers to package applications and their dependencies into portable containers. By mastering Docker commands, managing containers, images, networks, volumes, and using Docker Compose, you can easily build, test, and deploy multi-container applications.
